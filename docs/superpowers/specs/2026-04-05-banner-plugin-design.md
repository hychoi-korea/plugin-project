# Banner 생성기 Figma 플러그인 설계

**날짜:** 2026-04-05
**대상:** 디자이너 및 콘텐츠 마케터
**목적:** 피그마 배너 템플릿에 AI 기반으로 카피·이미지·컬러를 자동으로 채워주는 플러그인

---

## 1. 전체 아키텍처

```
┌─────────────────────────────────────────┐
│  Figma Plugin UI (ui.html)              │
│  - 단일 스크롤 패널                      │
│  - Claude / Gemini API 호출 (fetch)     │
└────────────────┬────────────────────────┘
                 │ postMessage
┌────────────────▼────────────────────────┐
│  Plugin Main Thread (code.ts)           │
│  - Figma API 접근 (레이어 탐색/수정)    │
│  - clientStorage (API 키 저장/조회)     │
│  - 선택된 노드 분석 및 콘텐츠 적용      │
└────────────────┬────────────────────────┘
        ┌────────┴────────┐
        ▼                 ▼
┌──────────────┐  ┌──────────────┐
│  Claude API  │  │  Gemini API  │
│  (카피 생성/ │  │  (이미지 생성)│
│   다듬기)    │  │              │
└──────────────┘  └──────────────┘
```

**핵심 원칙:**
- API 호출은 UI 스레드(브라우저 환경)에서 수행 — 메인 스레드는 `fetch` 불가
- 두 스레드 간 통신은 `postMessage` / `onmessage`로만
- `manifest.json`의 `networkAccess.allowedDomains`에 Claude(`api.anthropic.com`), Gemini(`generativelanguage.googleapis.com`) 도메인 추가

## 3. 레이어 매핑 로직

피그마 프레임 내 레이어를 이름 기반으로 재귀 탐색 (`findAll`)하여 자동 매핑.

| 레이어명 | 매핑 내용 | 제약 |
|---|---|---|
| `#main_copy_01` | 메인 카피 앞부분 (한 문장을 나눈 것) | 최대 8자 |
| `#main_copy_02` | 메인 카피 뒷부분 (한 문장을 나눈 것) | 최대 8자 |
| `#main_copy_03` | 메인 카피 전체 | 최대 16자 |
| `#main_copy_04` | 메인 카피 요약 (m_band용) | 최대 11자 |
| `#sub_copy` | 서브 카피 | 최대 16자 |
| `image_main` | 메인 이미지 | — |
| `image_sub_01` | 서브 이미지 1 (없으면 스킵) | — |
| `image_sub_02` | 서브 이미지 2 (없으면 스킵) | — |
| `badge` | 뱃지 컴포넌트 | 컴포넌트명 매칭 |

**분리 로직:**
- `#main_copy_01` / `#main_copy_02`: 메인 카피 16자를 어절 단위로 앞/뒤 분리. 어절 경계가 8자를 넘으면 Claude가 자연스러운 위치로 분리
- `image_sub_01/02`: 해당 레이어가 없는 프레임은 자동 스킵
- 섹션 선택 시 하위 프레임 모두 순회하며 동일 로직 적용

**적용 범위:**
- 섹션 선택 → 하위 모든 프레임에 적용
- 프레임 선택 → 해당 프레임에만 적용

## 4. API 연동

### Claude API (카피 생성/다듬기)

**요청:**
```json
{
  "mode": "generate" | "refine",
  "input": "키워드 or 초안",
  "constraints": {
    "main_copy_max": 16,
    "sub_copy_max": 16,
    "split_at": 8,
    "m_band_max": 11
  }
}
```

**응답:**
```json
{
  "main_copy_01": "앞부분 (최대 8자)",
  "main_copy_02": "뒷부분 (최대 8자)",
  "main_copy_03": "전체 (최대 16자)",
  "main_copy_04": "요약 (최대 11자)",
  "sub_copy": "서브 카피 (최대 16자)"
}
```

- 글자수 초과 시 자동 재요청 (최대 2회 retry)
- 키워드 입력 시 → 생성 모드
- 초안 입력 후 AI 버튼 → 다듬기 모드

### Gemini API (이미지 생성)

- 입력: 제품 이미지 1~3장 (base64) + system instructions (PDF 정의 내용) + 배너 사이즈 컨텍스트
- 출력: 라이프스타일 연출 이미지 1장
- 생성 중 로딩 스피너 표시
- 생성 완료 후 플러그인 내 미리보기 제공
- 확정 시 `image_main` 레이어에 적용

### 배경 컬러 추천 (클라이언트 사이드)

- Canvas API로 이미지 픽셀 샘플링 → 키컬러 3개 추출 (빈도 높은 색상)
- 키컬러 기반 보색 계산 → 이미지를 돋보이게 할 컬러 3개 추천
- API 호출 없이 브라우저에서 처리 (빠름, 비용 없음)
- 아이디어스 팔레트(deep/vivid/light/pastel, 각 13색) 전체 표시 및 자유 선택
- 선택 후 컬러 피커 / HEX 입력으로 자유 수정
- 변경 즉시 캔버스 실시간 반영

## 5. API 키 관리

- `figma.clientStorage`에 저장 (해당 컴퓨터의 Figma 앱에만 저장)
- 최초 실행 시 입력 모달 표시
- 이후 실행 시 저장된 값 자동 사용
- 설정 섹션에서 언제든 수정 가능

## 7. 파일 구조

```
new plugin/
├── manifest.json          # networkAccess 도메인 추가
├── code.ts                # 메인 스레드
│   ├── 선택 노드 분석 (섹션/프레임 판별)
│   ├── 레이어 재귀 탐색 및 매핑 (findAll)
│   ├── 뱃지 컴포넌트 자동 탐색
│   ├── clientStorage API 키 저장/조회
│   └── 콘텐츠 적용 (텍스트/이미지/컬러/뱃지)
├── ui.html                # UI 스레드 진입점
├── src/
│   ├── claude.ts          # Claude API 클라이언트
│   ├── gemini.ts          # Gemini API 클라이언트
│   ├── colorExtractor.ts  # 키컬러 추출 + 보색 계산 (Canvas API)
│   └── layerMapper.ts     # 레이어명 매핑 로직
└── package.json
```
